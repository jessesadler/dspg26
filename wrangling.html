4  Wrangling data in the tidyverse

When working with a data frame, you will usually want to modify it in some way. You may want to make some calculations based on different columns in the data or calculate some summary statistics. You may only need a subset of the data, either of the rows or columns. You might want to split the data into separate data frames or join together related data frames. Or you may need to clean up the data, changing column names, modifying values in different ways, or changing the shape of the data frame before doing your analysis and visualization. These are some of the more common tasks of data wrangling.1

These tasks can be broken into three main categories:

  1. Manipulating rows and columns.
  2. Joining together related data frames.
  3. Modifying the structure of a data frame.

4.1 Pipes

Before getting into the details of how to go about doing these data wrangling tasks using the tidyverse, we should introduce a tool that will make this process of data cleaning much easier to write and more legible to read, namely the pipe (|>).2 In doing almost any data wrangling task, you will want to perform multiple manipulations. You may want to remove some columns, and then remove rows with missing data, and then calculate the frequency of one or more column, and then calculate some summary statistics.

The pipe is the and then. It links together multiple functions together, taking the output from the left of the pipe and moving it to the function on the right. Using the pipe will look something like this, and remember you can think of it as and then:

data scientists |>
  collect data |>
  explore the data |>
  read about the context of the data |>
  analyze the data |>
  visualize the data |>
  publish an analysis

In this analogy data scientists take the place of the data and the various actions (verbs) are the functions.

4.2 1. Manipulating rows and columns with dplyr

The core set of dplyr functions are designed to work with the pipe. They take in a data frame as their input in the first argument (named .data); subsequent arguments include the (unquoted) column names that are operated on; and the output is a data frame, which can be piped into another function. You can think of these functions as verbs to manipulate rows and columns. The core functions are:

Function Description
select() Subset columns.
filter() Subset rows by conditions based on data in columns.
mutate() Create new columns using information from other columns.
group_by() Group the data by values in column such as by village or by sex.
summarize() Aggregate the data, usually on grouped data, to create summary tables.
arrange() Arrange the order of rows by values in columns. Often used with desc() to reverse default order.
  • To read more about the core dplyr functions, see R for Data Science: Data Transformation.
  • For even more practice, check out the Data Carpentries, R for Social Science episode on Data Wrangling with dplyr.

Let’s look at an example using the SAFI_clean.csv data from the R for Social Science lesson.

library(tidyverse)
# Read the data without further specifications
interviews <- read_csv("data-raw/SAFI_clean.csv", na = "NULL")
interviews |>
1  select(village, no_membrs:liv_count) |>
2  filter(!is.na(memb_assoc)) |>
3  mutate(people_per_room = no_membrs / rooms) |>
4  group_by(village, memb_assoc) |>
5  summarise(mean_per_room = mean(people_per_room),
6            n = n(),
7            .groups = "drop") |>
8  arrange(desc(mean_per_room))

5

6

7

8
Arrange the rows by the average people per room in descending order.

4.3 2. Joining together related data frames

A common data wrangling task is to join together related data frames. This situation can come about by the desire split up a dataset into multiple datasets to keep them clean and tidy or the raw data may already be separated into different data frames. Joins make it possible to work with multiple datasets at a time.

For instance, in collecting field data, you might have a spreadsheet that notes the dates you conducted the interviews and a separate spreadsheet for the data collected. The two data frames can be reconnected as long as they share a column that identifies each row. In interviews this is the key_ID column.

# Split the data into to data frames
interview_dates <- interviews |>
  select(key_ID, interview_date)

interviews <- interviews |>
  select(!interview_date)

There are four main join functions. All take two data frames and join them by a column that is shared by both data frames. The names of the column to join by do not need to be the same, but it this case they are, namely "key_ID". The four functions differ in how they join the data frames.

Function Description
left_join() Keep all observations in the first (left) data frame.
right_join() Keep all observations in the second (right) data frame.
full_join() Keep all observations from both data frames.
inner_join() Only keep observations that are in both data frames, dropping rows that do not have a match.

Generally, it is best to default to left_join(). In this case, both data frames have all of the key_IDs, so it does not matter which one you use.

left_join(interview_dates, interviews, by = "key_ID")

4.4 3. Modifying the structure of a data frame

A third task you may face is needing to alter the structure of your data to make it tidy. This is the specialty of the tidyr package. The principles of tidy data are:

  1. Each variable is a column; each column is a variable.
  2. Each observation is a row; each row is an observation.
  3. Each value is a cell; each cell is a single value.

Data is not always tidy; sometimes for good reason. For instance, in collecting data it might be easier to place more than one value in single column such as the items_owned and months_lack_food columns in SAFI_clean.csv. However, this structure for the data makes it difficult to know which items each household owned, how many, and what months they lacked food. tidyr provides tools to change the structure of the data into something more useful.

Given the structure of the data in SAFI_clean.csv, if you wanted to know more about the months in which households lacked food, it is necessary to separate out the data either into separate columns or rows. This example makes the data longer by separating the months into rows to look at the months that households lacked food by village.

lack_food <- interviews |> 
1  select(key_ID, village, months_lack_food) |>
2  separate_longer_delim(months_lack_food, delim = ";")
